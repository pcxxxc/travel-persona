# 架构决策记录 (Architecture Decision Records)

本目录存放旅格项目的重大技术决策记录。每个决策对应一个文件，命名格式：`NNNN-short-title.md`。

## ADR 格式

每个 ADR 文件必须包含以下段落：

```
# 标题

## 状态
已采纳 | 已废弃 | 已替代 | 待定

## 背景 (Context
为什么需要这个决策？当前状况和驱动因素。

## 决策 (Decision
我们做了什么决定。

## 备选方案 (Alternatives
考虑过但未采纳的方案及原因。

## 后果 (Consequences
这个决策带来的正面和负面影响。

## 迁移 (Migration
如何从当前状态过渡到新方案（如适用）。

## 验证 (Verification
如何确认决策已被正确实施。
```
