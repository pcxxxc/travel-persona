# 0003: 冻结静态 Demo 页面

## 状态

已采纳

## 背景

`public-site/travel-persona/index.html` 是项目早期的单页交互原型，所有功能（人格分析、手账、成长Demo、推荐、路线、地图、API状态）纵向堆叠在一个长页面中。

总纲 2.2 P0 缺口明确指出："单页纵向堆叠全部能力，像项目汇报页，不像可重复使用的旅行产品。"

总纲 4.1 要求："生产版本 MUST 使用多视图应用结构，不再把人格、手账、Demo、路线、地图和 API 全部纵向堆在首页。"

总纲 18.3 禁止的捷径第一条："不得在当前单页继续追加更多 section 代替信息架构重构。"

## 决策

**`public-site/travel-persona/index.html` 自 2026-07-11 起标记为 legacy/reference。**

规则：
1. 不得在该页面追加新功能、新 section 或新交互
2. 仅允许修复影响现有功能的 critical bug
3. 新的前端开发在多视图应用结构中进行（Phase 3）
4. 该页面保留为交互原型和数据验证台的参考

## 备选方案

1. **直接删除**：否决——页面包含可复用的交互思路和数据验证逻辑，保留为参考。
2. **继续在现有页面迭代**：否决——总纲明确禁止，且长单页结构无法支持总纲定义的多视图信息架构。

## 后果

**正面：**
- 防止在错误架构上继续投入
- 团队精力转向正确的多视图应用结构
- 旧页面可作为"从哪里来"的参照

**负面：**
- 短期内没有新的前端可展示，需要等到 Phase 3 才有新的用户界面
- 旧页面的部分交互逻辑可能需要在 Phase 3 重新实现

## 迁移

- Phase 0：添加 legacy 标记注释和 meta 标签
- Phase 3：在 `apps/web/` 中实现新的多路由应用
- Phase 6 发布后：可选将 `public-site/` 整体归档到 `legacy/demo-2026-07/`

## 验证

- `public-site/travel-persona/index.html` 包含 legacy 标记
- 后续 commit 中该文件无新增 section 或功能代码
